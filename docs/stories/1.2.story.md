# Story 1.2 : Persistent Root Layout & Global Audio Store
**Epic :** 1. Core Experience & Studio UI

## User Story
En tant qu'utilisateur, je veux que l'ambiance visuelle (fond sombre) et sonore (lecteur) soit persistante afin de ne pas subir de coupure lors de ma navigation.

## Acceptance Criteria
1. Le fichier `layout.tsx` contient un wrapper global (`StudioProvider`).
2. Le store Zustand est capable de maintenir l'état `isPlaying` et `currentTrackId`.
3. Un changement de route ne provoque pas le rechargement du composant audio.

## Tasks
- [ ] Créer le store Zustand `useAudioStore` dans `/store/useAudioStore.ts`.
- [ ] Implémenter le composant `StudioLayout` avec l'effet de grain visuel.
- [ ] Ajouter une balise `<audio>` invisible dans le layout racine pour gérer le flux sonore.
- [ ] Vérifier la persistance en changeant de page de test.
